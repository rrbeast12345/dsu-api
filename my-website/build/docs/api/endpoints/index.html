API Endpoints Reference

This document provides a complete reference for all API endpoints in the DSU API system.

Base URL​

  • Development: http://localhost:8000
  • Production: https://dsu-api.onrender.com

Authentication Endpoints​

POST /login​

Authenticate a user with username and password.

Request Body:

{
  "username": "string",
  "password": "string"
}

Response:

{
  "success": true,
  "id": "user_id_number"
}

POST /register​

Register a new user account.

Request Body:

{
  "id_number": "string",
  "name": "string", 
  "dob": "YYYY-MM-DD",
  "password": "string"
}

Response:

{
  "success": true,
  "person": {
    "id_number": "string",
    "name": "string",
    "dob": "string",
    "verified": false,
    "income_bracket": -1,
    "age": 25,
    "citizenship_status": false,
    "consents": [],
    "current_grants": [],
    "identifications": [],
    "file_ids": [],
    "payment_forms": [],
    "payment": [],
    "grant_ids": []
  }
}

POST /verify_identity​

Verify user identity with biometric data.

Request Body:

{
  "id_number": "string",
  "name": "string",
  "dob": "YYYY-MM-DD"
}

Response:

{
  "status": "verified|unverified|hidden",
  "biometrics": "match|non-match"
}

User Management Endpoints​

POST /get_user​

Retrieve user information by ID.

Parameters

  • id: "654"

Response:

{
  "id_number": "string",
  "name": "string",
  "dob": "string",
  "verified": true,
  "income_bracket": 3,
  "age": 25,
  "citizenship_status": true,
  "consents": ["scope1", "scope2"],
  "current_grants": ["grant1", "grant2"],
  "identifications": ["doc1", "doc2"],
  "file_ids": ["file1", "file2"],
  "payment_forms": ["form1"],
  "payment": [{"provider": "string", "number": "string"}],
  "grant_ids": ["grant_id1", "grant_id2"]
}

POST /delete user​

Delete a user account.

Request Body:

"user_id_string"

Response:

"user removed"

POST /verify user​

Verify a user account (admin function).

Request Body:

"user_id_string"

Response:

"user verified"

PUT /update_user​

Update user profile information. Parameters

  • id: "654"

Request Body:

{
  name: "sillybb",
  income_bracket: "1",
  dob: "2000-02-02",
  age: "25",
  citizenship_status: true
}

Response

{
  "success":true
}

PUT /set user​

Set specific user parameters.

Parameters:

  • id: User ID
  • parameter: Parameter name
  • value1: Parameter value (for values with lists, seperate values with commas)

POST /apply​

Apply for a specific grant.

Parameters:

  • id: User ID
  • grant: Grant name

Response:

{
  "success": true,
  "message": "Application submitted successfully|you are missing ID"
}

POST /get grants​

Get user's grant applications and status.

Request Body:

"user_id_string"

Response:

{
  "grants": [
    {
      "application id": "30",
      "grant": "Business Grant",
      "admin_approved": "true|false",
      "approved": "2023-01-01T00:00:00Z",
      "nextPayment": "2025-07-15T08:23:04Z",
      "application_status": "true",
      "description": "apply for business loan"
    }
  ]
}

GET /get all requirements​

Get all possible grant requirements across all grants.

Response:

{
  "requirements": [
    "Proof of Previous Employment",
    "Senior Certificate",
    "Business Statement",
    "Grades",
    "Bank Statement",
    "ID"
  ]
}

Document Management Endpoints​

POST /enter_identification​

Upload identification documents.

Parameters:

  • id: User ID
  • identification: Document type
  • image: File upload Request Body:
{
"image": "File upload"
}

Response:

{
  "success": true
}

GET /recieve_docs​

Retrieve user documents.

Parameters:

  • id: User ID

Response:

{
  "documents": [
    {
      "files":["46.jpg"],
      "docs": ["Bank Statement"]
    }
  ]
}

Consent Management Endpoints​

GET /consent_scopes​

Get available consent scopes.

Response:

{
  "scope": [
    "Send photos",
    "See ID",
    "Use my data to support application"
  ]
}

GET /get_consent​

Get user consent status.

Parameters:

  • id: User ID

Response:

{
  "scope": ["scope1", "scope2"]
}

POST /record_consent​

Record user consent.

Request Body:

{
  "id_number": "65434",
  "consent_given": true,
  "scope": [
    "See ID"
  ],
  "remove_scope": [
    "Send photos",
    "Use my data to support application"
  ]
}

Response:

{
    "form": {
        "id_number": "123",
        "consent_given": true,
        "scope": [
            "Send photos",
            "Use my data to support application"
        ],
        "remove_scope": [
            "See ID"
        ],
        "birthday": "2025-07-08T08:37:14Z"
    },
    "id": "38"
}

POST /retrieve_consent_form​

Get consent form for user.

Request Body:

"user_id_string"

Response

{
  "id_number": "12345",
  "consent_given": true,
  "scope": [
    "Send photos",
    "See ID",
    "Message my business"
  ],
  "remove_scope": [
    "Use my data to support application"
  ],
  "date": "2025-06-26T10:30:55Z"
}

Payment Management Endpoints​

POST /set_payment​

Set payment information.

Request Body:

{
  "id_number": "string",
  "wallet_provider": "string",
  "wallet_number": "string"
}

Response

{
  "form": {
    "id_number": "123",
    "wallet_provider": "amex",
    "wallet_number": "124",
    "birthday": "2025-07-08T08:39:41Z"
  },
  "id": "7"
}

POST /retrieve_pay_form​

Get payment form for user.

Request Body:

"user_id_string"

Response

{
  "id_number": "123",
  "wallet_provider": "amex",
  "wallet_number": "124",
  "birthday": "2025-07-08T08:41:12Z"
}

GET /get_payment​

Get user's payment details.

Parameters:

  • ids: User ID

Response:

{
  "payment_info": [
    {
      "provider": "string",
      "number": "string",
      "date_added": "2023-01-01T00:00:00Z"
    }
  ]
}

POST /admin/approve_grant​

Approve a grant application.

Request Body:

{
  "user_id": "456",
  "grant_id": "27"
}

response

{"success":true,"message":"Grant approved."}

POST /admin/disapprove_grant​

Disapprove a grant application.

Request Body:

{
  "user_id": "string", 
  "grant_id": "string"
}

Error Responses​

All endpoints may return error responses in the following format:

{
  "success": false,
  "error": "Error description"
}

Common error scenarios:

  • User not found
  • Invalid credentials
  • Missing required fields
  • File upload failures
  • Database errors
